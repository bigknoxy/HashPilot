The Problem

AI agents edit code blind

No IDE, no visual location, no working memory of file structure. Yet tools expect agents to edit like humans — counting lines and guessing text.

Line Counting

"Replace line 42" breaks when one import shifts every line. +3 retries, corrupted file.

Fuzzy Find-Replace

Guess the right text to match — or match the wrong occurrence. Silent corruption, wasted tokens.

Race Conditions

Two edits interleave. Invalid syntax, agent confusion, no recovery.

The Solution

Cryptographic certainty for every edit

🔐

SHA-256 Anchors

Target content by its cryptographic fingerprint. If the hash matches, you're editing the right content. No ambiguity.

For agents — never edit the wrong content
⚡

3-Tier Routing

AST → Hash → Diff. Auto-selects the best strategy per file + operation. One command, right tool every time.

For agents — one shot, every time
🔄

Stale Recovery

Detects out-of-date edits (file changed since read) and re-applies transparently. No silent corruption.

For agents — survive race conditions
🎯

Intents (M5)

"Add a parameter to processData" → discovers all call sites, plans the edit, executes every step.

For agents — multi-file refactors in one shot
✅

Verification Bundling

Run formatter + linter + typechecker + tests in one command. Optional revert on failure.

For developers — know it works
📋

Provenance

provenance query src/utils.ts shows edit history per line. Like git blame for agent edits.

For teams — audit trail for AI changes

Quick Start

Install in one command

curl -fsSL https://raw.githubusercontent.com/bigknoxy/HashPilot/main/scripts/install.sh | sh
Requires Bun 1.2+ — installs CLI, configures Claude Code, OpenCode, and Pi adapters automatically.

Your First Edit

# 1. Read a file — get its content hash
structured-edit read-many src/main.ts

# 2. Edit by hash — target the exact content
HASH="abc123..."  # from read-many output
structured-edit replace-hash src/main.ts "$HASH" "  port: 8080" --range 5:5

# 3. Verify nothing broke
structured-edit verify-changes src/main.ts --auto-detect
Architecture

The 3-tier routing model

┌─────────────────┐ │ Your Edit │ └────────┬────────┘ │ ▼ ┌──────────────────────────┐ │ 1. AST Route │ ◄── tree-sitter syntax edits │ TS/TSX/JS/Python/Go/Rust│ rename-symbol, replace-body └────────────┬─────────────┘ │ unsupported ▼ ┌──────────────────────────┐ │ 2. Hash Route │ ◄── SHA-256 anchored replacement │ (any file) │ stale-anchor detection └────────────┬─────────────┘ │ no hash ▼ ┌──────────────────────────┐ │ 3. Diff Route │ ◄── LCS-based search-and-replace │ (fallback) │ fuzzy matching └──────────────────────────┘

The router auto-selects. route-edit tries AST → Hash → Diff. Every route records telemetry and provenance.

Integrations

Pi

Native extension with 7 custom tools + /hp slash command.
